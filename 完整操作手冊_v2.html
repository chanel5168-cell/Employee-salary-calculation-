1. 系統簡介與核心功能

這是一套建置在 LINE LIFF + Firebase 上的員工薪資計算系統，適合中小型企業、連鎖店、多廠區或跨區域管理。

🎯 核心功能

👥 員工管理月薪/時薪制、保險、眷屬、自提勞退都支援
📅 每日計時論件、計時、加班、請假一頁搞定，可附照片
💰 月薪結算含勞健保、加班費、特休折算、請假扣款
📊 分析圖表月度趨勢、廠別占比、加班排行
📦 多種匯出Excel 薪資單、會計傳票、照片 ZIP、特休稽核
🌴 特休追蹤依勞基法自動計算，稽核表可給勞保局查
🏭 多廠別隔離香港/台北互不可見，資料安全分離
☁ 雲端同步Firebase 即時同步，多裝置協作

📱 支援的使用方式

  • 網頁瀏覽器：Chrome / Edge / Safari / Firefox（手機與電腦都可）
  • LINE App：透過 LIFF 直接在 LINE 裡使用

2. 🚀 3 分鐘快速上手

💡 第一次使用的人請照順序做：
1

登入管理者帳號
內建帳號 admin，密碼由系統設定（第一次請向建置人員詢問）
3
到「設定」頁建立使用者
為主管、會計、廠別負責人建立帳號，指定角色與廠別範圍

員工編號、姓名、薪資、工作地點、勞健保等基本資料

3. 👥 四種角色與權限

👑 總管理者 (super)

• 最高權限，看所有資料
• 可管理所有帳號（含其他總管理者）
• 可修改 LIFF 設定、勞健保費率
• 可匯出所有報表
• 可執行所有清除、同步操作

🔧 管理者 (admin)

• 預設可看所有資料
• 可指定限定廠別（香港/台北資料隔離）
• 可新增、編輯員工與薪資
• 不能修改總管理者帳號
• 可匯出限定範圍的報表

🏭 廠別主管 (factory)

• 只看指定廠別的員工、薪資
• 可編輯該廠員工資料
• 可匯出該廠的做帳報表
• 適合會計人員使用
• 看不到「設定」頁

👤 員工 (employee)

• 只看自己的薪資與出勤
• 無法編輯任何資料（唯讀）
• 無匯出權限
• 不能看「分析」、「匯出」、「設定」
• 適合一般員工查詢薪資用
⚠ 權限階層規則：
• 只有較高權重的角色才能建立、編輯、刪除較低權重的帳號
• 權重順序：👑 super > 🔧 admin > 🏭 factory > 👤 employee
• 總管理者例外：可編輯其他總管理者（管理人員交接用）

🏭 廠別範圍設定（香港/台北資料隔離）

當您想讓「香港場管理者」看不到「台北場」的員工資料時，可以這樣做：

  1. 登入總管理者 → 進入「設定」頁
  2. 在「使用者管理」新增使用者
  3. 角色選「🔧 管理者」
  4. 勾選廠別時只勾香港相關廠別（例：香港倉、香港廠）
  5. 儲存後，該管理者登入就只看得到香港員工與薪資
💡 會計人員怎麼設定？
會計人員通常建議用「🏭 廠別主管」角色，並勾選其負責的廠別。這樣他可以：
✓ 看到負責廠別的員工、薪資
✓ 匯出 Excel、會計做帳表、照片 ZIP、特休稽核表
✗ 看不到其他廠別的資料，符合資料隔離

4. 📝 七個頁籤詳細操作

① 員工頁

建立與維護員工基本資料，包括薪資設定、勞健保、工作地點。

🆕 新增員工的欄位

欄位說明
員工編號選填，例 E001。方便內部識別
姓名 *必填
到職日用於特休計算（勞基法 §38）
投保日勞健保起算日
月薪（底薪）月薪制填這欄；時薪制可填 0
時薪時薪制必填；月薪制可留空（系統自動 = 月薪÷30÷8）
月投保薪資勞健保依此計算
工作地點廠別範圍限制的關鍵欄位，一定要填
健保眷口數0~3 口，健保費 = 投保薪資×1.655%×(1+眷口數)
自提勞退 %0~6%，員工自願從薪資扣除

② 每日頁

登記員工每天的工作內容（論件、計時、加班、請假）。

四種記錄類型

  • 📦 論件：選產品、填數量與單價，自動計算金額
  • ⏰ 計時：填起迄時間、扣除休息，系統自動算時數 × 時薪
  • 🔥 加班：平日加班（1.34×）、假日加班（1.67×）依勞基法自動計算
  • 🌴 請假：選類別（特休/病假/事假…），自動換算扣款
📷 拍照功能：論件、計時都可以附照片（例如現場產出照、工時證明），照片會自動壓縮節省空間。
⏰ 時數規則（系統自動處理）：
• 不足 30 分鐘不計
• 30 分鐘為單位捨去
• 範例：45 分 → 30 分；59 分 → 30 分；60 分 → 60 分

③ 月薪頁

彙整當月所有每日記錄，計算最終薪資。

薪資組成

應發合計 = 底薪 + 論件收入 + 計時收入 + 特休折算 + 平日加班費 + 假日加班費 + 獎金 - 請假扣款 勞健保扣除 = 勞保費 + 健保費 × (1 + 眷口數) 實發金額 = 應發合計 - 勞健保扣除 - 其他扣款
💡 計算後記得儲存：系統會產生「月薪記錄」存到 records。儲存後才會在匯出頁看到。

④ 分析頁（管理者專用）

視覺化報表 + 會計人員做帳匯出。

4 張圖表

  1. 📈 近 6 個月總薪資趨勢（折線圖）：應發 / 實發雙線
  2. 🥧 廠別薪資占比（甜甜圈圖）：看哪個廠薪資支出最多
  3. 📊 加班時數 TOP 10（橫向柱狀圖）：找出加班最多的人
  4. 💳 摘要卡：員工人次、應發、勞健保、實發、加班總時數

5 種匯出（都會依您的可管理廠別過濾）

按鈕用途
📥 匯出分析 Excel含摘要、廠別小計、月薪明細三張工作表
💼 匯出會計做帳表含借方/貸方分錄，會計可直接寫傳票
📊 下載完整月薪 Excel含摘要、月薪明細、出勤明細多張工作表
📷 下載照片 ZIP所有計時照片打包（檔名含日期_員工_品名）
🌴 特休稽核表勞保局格式，總覽 + 每位員工個別明細

⑤ 匯出頁

原始匯出功能（和分析頁部分重疊）+ 資料管理工具。

📥 匯出類

  • 下載 Excel 檔案（月薪總表）
  • 📷 下載所有照片 ZIP
  • 🌴 特休請假稽核表

📊 統計摘要

顯示各月份總覽，可點展開看明細。

🛠 資料管理

  • 備份 JSON：全部資料匯出為 .json 備份檔
  • 🔐 還原備份：從 .json 檔還原（需管理者密碼）
  • 🔐 清除全部：清除本機資料（需管理者密碼）
  • 🧹 清理重複記錄：移除重複的月薪/每日記錄
  • 🔧 修正時數/加班：重新計算金額
  • 📷 清理 30 天前舊照片：節省 Firebase 空間（手動執行）
  • 🔐 ☁ 強制重設雲端：清空 Firestore 資料
  • 🔄 從雲端強制同步：把雲端資料覆蓋本機

⑥ 特休頁

依勞基法 §38 自動計算每位員工的特休額度。

年資特休天數
6 個月~1 年3 天
1~2 年7 天
2~3 年10 天
3~5 年14 天
5~10 年15 天
10 年以上每滿 1 年加 1 天，上限 30 天
💡 時薪制員工：依比例計算，公式 = 全時特休 × (每週工時 ÷ 40)

⑦ 設定頁（僅管理者）

系統最上方是 📱 LINE LIFF 設定，接著是：

  • 👥 使用者管理：新增/編輯/刪除帳號
  • 🎌 國定假日管理：新增假日，影響月薪頁出勤紅字
  • 💰 台灣勞健保參數：2026 年的勞保 2.42%、健保 1.655%（法規異動時才改）

5. 💼 常見工作情境

情境 A：新員工報到

  1. 到「員工」頁 → 填寫基本資料 + 薪資設定 + 工作地點 → 儲存
  2. 若需要該員工自行登入查薪資 → 到「設定 → 使用者管理」新增帳號，角色選「👤 員工」，對應員工選剛建立的
  3. 告訴員工帳號密碼 → 完成

情境 B：每月薪資結算（會計流程）

  1. 確認當月所有「每日」記錄都完整（可從匯出頁「統計摘要」檢查）
  2. 到「月薪」頁 → 選員工 → 選月份 → 系統自動算出應發 → 填獎金/扣款 → 儲存
  3. 重複上一步處理每位員工（或用「全部月薪制員工一起算」）
  4. 到「分析」頁 → 選該月份 → 按「💼 匯出會計做帳表」
  5. 打開 Excel → 把分錄寫進會計系統的傳票

情境 C：香港/台北資料隔離

  1. 總管理者登入 → 設定 → 使用者管理
  2. 新增「香港場管理者」：角色 = 🔧 管理者，勾選香港相關廠別
  3. 新增「台北場管理者」：角色 = 🔧 管理者，勾選台北相關廠別
  4. 分別把帳密告知各場主管
  5. 兩人登入後看到的員工、薪資、圖表、匯出都只有自己廠別

情境 D：勞保局稽核

  1. 到「分析」或「匯出」頁
  2. 點「🌴 特休稽核表」
  3. 下載 Excel，含「總覽」工作表 + 每位員工個別明細工作表
  4. 列印或直接 email 給勞保局

情境 E：換手機/換電腦

  1. 舊裝置：到「匯出 → 資料管理」→ 按「備份 JSON」下載備份檔
  2. 新裝置：開啟網頁 → 登入 → 到「匯出 → 資料管理」→ 按「🔐 還原備份」
  3. 或更簡單：新裝置不用還原，登入後系統會自動從 Firebase 同步所有資料

6. ☁ 如何放上雲端（GitHub Pages + LINE LIFF）

整個部署流程約 30 分鐘，都是免費的
GitHub Pages、Firebase、LINE LIFF、LINE Developers 通通終身免費

第一部分：GitHub Pages（網頁主機）

1
註冊 GitHub
到 https://github.com，用 email 註冊（完全免費）
2
建立倉庫（Repository）
登入後右上角 + → New repository
• Repository name 填：salary-app（或其他名稱）
• 設為 Public（GitHub Pages 免費版需要公開）
• 勾選 Add a README file
• 按 Create repository
3
上傳 index.html
在倉庫首頁 → Add file → Upload files → 拖入 index.html → 下方 Commit message 寫「首次上傳」→ 按綠色 Commit changes
4
啟用 GitHub Pages
倉庫頁面上方 Settings → 左側 Pages → Source 選 main branch → Save
5
等待約 1~2 分鐘 → 取得網址
頁面會顯示 https://您的帳號.github.io/salary-app/，點開就是您的系統網頁！

第二部分：Firebase（雲端資料庫）

1
註冊 Firebase
到 Firebase Console，用 Google 帳號登入 → 點 Add project 建立專案
2
啟用 Firestore
左側選單 → Firestore Database → Create database → 選「測試模式」(開啟 30 天臨時規則) → 選區域（建議 asia-east1）→ Enable
3
取得 Firebase 設定資訊
專案首頁 → Project settings（齒輪圖示）→ 下方「您的應用程式」→ 選「Web 應用程式 </>」→ 註冊 app → 複製 firebaseConfig 這段程式碼
4
把 firebaseConfig 貼到 index.html
打開 index.html → 搜尋 firebaseConfig → 用您的值取代裡面的 apiKey、projectId 等 → 重新上傳到 GitHub
🚨 重要：Firestore 測試模式 30 天後會鎖死！請在到期前到 Firestore → 規則頁面，改成永久規則。系統內會有倒數提醒橫幅（管理者登入時顯示），點橫幅可看完整設定步驟。

第三部分：LINE LIFF（在 LINE 裡開啟）

1
註冊 LINE Developers
到 LINE Developers，用 LINE 帳號登入
2
建立 Provider
點 Create a new provider → 輸入公司/組織名稱
3
建立 LINE Login Channel
在 Provider 下 → Create a new channel → 選 LINE Login
• Channel name：薪資系統
• App types：勾 Web app
• 填完表單 → 建立
4
新增 LIFF App
進入 Channel → LIFF 分頁 → Add
• LIFF app name：薪資系統
• Size：Full
• Endpoint URL：貼您的 GitHub Pages 網址
• Scope：勾 profile、openid
• 建立後會取得 LIFF ID（格式如 1234567890-AbCdEfGh）
5
在系統裡設定 LIFF ID
打開您的系統網頁 → 登入總管理者 → 設定頁最上方「📱 LINE LIFF 設定」→ 貼入 LIFF ID → 按「儲存並重新初始化」
6
在 LINE 裡測試
LIFF URL 為：https://liff.line.me/您的LIFF_ID
把這個網址貼到 LINE 聊天 → 點擊 → 在 LINE 內開啟系統
✅ 部署完成後，三個網址都可以用：
• 電腦瀏覽器：https://您的帳號.github.io/salary-app/
• 手機瀏覽器：同上
• LINE 內：https://liff.line.me/您的LIFF_ID

7. 🔧 如何修改系統

無論是想調整文字、改按鈕顏色、加新功能，都是修改同一個 index.html 檔案。

方法 A：直接在 GitHub 網頁編輯（簡單改動適用）

1
進入倉庫首頁
到 https://github.com/您的帳號/salary-app
2
點 index.html
進入檔案後，右上角有鉛筆圖示 🖊 或 Edit in place，點下去
3
編輯內容
用 Ctrl+F 搜尋要改的文字 → 修改
4
Commit changes
右上角 Commit changes → 寫說明 → 綠色按鈕 → 完成
⚠ 網頁編輯器的限制：本系統 index.html 超過 8000 行，GitHub 網頁編輯器可能會卡頓或有卷軸問題。建議簡單改動用此法，大範圍改動用方法 B。

方法 B：下載到電腦 → 修改 → 上傳（推薦）

1
下載最新版
到 GitHub 倉庫首頁 → 綠色 <> Code 按鈕 → Download ZIP → 解壓縮到電腦
2
用編輯器打開 index.html
推薦：VS Code（免費）、Notepad++、甚至 Windows 內建的記事本都可以
3
修改並儲存
找到要改的地方 → 修改 → Ctrl+S 儲存
4
本機測試
雙擊 index.html 用瀏覽器打開 → 確認改動正確
5
上傳到 GitHub
倉庫首頁 → Add file → Upload files → 拖入新版 index.html → Commit message 寫「更新 XXX 功能」→ Commit changes
6
等 1~2 分鐘 → 強制重新整理
打開系統網頁 → Ctrl + F5（強制重整、清快取）→ 確認新版上線

方法 C：用 Cowork / Claude 幫您改（最輕鬆）

如果要改的功能比較複雜，可以把整份 index.html 上傳到 Cowork → 告訴 Claude 您要改什麼 → Claude 會直接幫您改好 → 下載後依方法 B 步驟 5 上傳。

常見修改範例

🎨 改變主色調（綠色改其他顏色）

搜尋 #06C755（LINE 綠），把它換成您想要的顏色代碼（如 #3b82f6 藍、#f59e0b 橘）。

💰 修改勞健保費率

不用改程式！到「設定」頁下方「台灣勞健保參數」直接改數值即可。

📝 新增工作地點/廠別

不用改程式！到員工頁 → 工作地點欄位旁的齒輪 ⚙ → 管理工作地點 → 新增。

📅 新增國定假日

不用改程式！到「設定 → 國定假日管理」 → 新增日期。

🌴 調整特休規則

搜尋 function calcAnnualLeave，該函式內有勞基法的天數對照，可以依公司規定調整（通常不建議改，依法就好）。

🔐 修改管理者密碼

搜尋 const ADMIN_PASSWORD，改成新密碼。

🕐 調整 30 天照片保留期

搜尋 PHOTO_RETENTION_DAYS = 30，改成您要的天數（例 60、90）。

8. 🛠 維護與常見問題

Q1：我新增了員工，但另一部電腦看不到？

  • 確認兩邊都有連上網
  • 左上角同步狀態應該顯示「☁ 已同步」
  • 若顯示錯誤，到「匯出 → 資料管理」按「🔄 從雲端強制同步」

Q2：員工看不到自己的薪資？

  • 確認該員工帳號的「對應員工」有設對（不是空的）
  • 確認員工的工作地點有填
  • 確認月薪頁已經儲存過該月資料

Q3：會計人員匯出的報表是全部員工，不是只有他負責的廠別？

  • 會計人員的角色是否是「🏭 廠別主管」？
  • 有沒有勾選正確的廠別？
  • 員工資料的「工作地點」有沒有填？（沒填廠別的員工不會被匯出）

Q4：Firebase 額度快滿了？

  • 最常見原因：計時照片太多
  • 到「匯出 → 資料管理」按「📷 清理 30 天前舊照片」
  • 系統也會每 24 小時自動清理一次
  • 若還是滿，可以在程式裡把 PHOTO_RETENTION_DAYS 改小（如 14 或 7）

Q5：不小心按了「清除全部」怎麼救？

  • 如果有「備份 JSON」→ 按「還原備份」選那份 .json 救回來
  • 如果沒備份，但雲端還有 → 按「🔄 從雲端強制同步」
  • 如果雲端也被清了 → 無法救回，所以 一定要定期備份！

Q6：忘記管理者密碼？

  • 程式內的固定密碼是在 ADMIN_PASSWORD 這個變數
  • 請聯絡原建置人員，或修改程式後重新上傳

Q7：頂部橫幅提醒 Firestore 規則快到期？

  • 點橫幅 → 彈出完整步驟說明
  • 按步驟到 Firebase Console 更新規則
  • 完成後回到系統，點橫幅 → 按「✅ 我已完成更新」
  • 橫幅就會永久消失

🗓 建議的維護排程

頻率工作
每月 1 次按「備份 JSON」下載完整備份
每月 1 次從匯出頁下載照片 ZIP 存檔（重要照片備援）
每年 1 次檢查勞健保費率是否依法調整
每年 1 次新增下一年度國定假日
依需要新增/離職員工處理（月結算前）

9. 🔐 安全性設定（Firestore 規則）

🚨 超級重要：新建 Firebase 專案的 Firestore 預設使用「測試模式」規則，30 天後會自動鎖死。請務必在到期前設定永久規則！

設定步驟

1
打開 Firebase Console
https://console.firebase.google.com → 選您的專案
2
進入 Firestore 規則頁
左側 Firestore Database → 上方頁籤 規則 (Rules)
3
刪除現有規則，貼上永久規則
全選現有內容 Delete → 貼上：
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /workspaces/default/{document=**} {
      allow read, write: if true;
    }
  }
}
4
按右上角「發布」(Publish)
等幾秒鐘 → 完成
5
回系統標記完成
登入系統 → 點到期提醒橫幅 → 「✅ 我已完成更新」→ 橫幅永久消失

規則說明

這段規則的意思：

  • workspaces/default/{document=**}：匹配 workspaces/default 底下所有文件（含子集合）
  • allow read, write: if true：允許讀寫
  • 其他未匹配的路徑：預設拒絕
💡 適用情境：小團隊內部使用，只要不公開 Firebase Project ID 就算足夠的安全（需要知道集合路徑 + SDK 才能存取）。
⚠ 如果要更嚴格的安全：可以整合 Firebase Authentication，讓規則變成 allow read, write: if request.auth != null（需要登入才能存取），但要改較多程式碼，請找工程師協助。
