員工薪資計算系統

會計操作手冊 · Accounting Manual
版本 v1.5 最後更新 2026-04-20 繁體中文

1. 系統概覽

本系統是一套專為小型公司（20 人以下）設計的薪資管理工具，透過 LINE LIFF 方式讓員工用手機輸入每日工作記錄，資料即時同步到雲端，會計／老闆在電腦上彙整、計算月薪並匯出 Excel 報表。

主要能力

類別功能
員工新增／編輯／刪除員工、設定底薪與時薪
產品建立品名、預設單價、參考照片
每日記錄論件計酬、計時計酬兩種模式，可拍照存證
月薪加班費、獎金、勞健保、扣款自動計算
匯出Excel 多工作表、照片 ZIP 打包
同步Firebase 雲端即時同步
備份JSON 匯出／匯入還原

角色分工

  • 會計／老闆（您）：建立員工、建立產品、查看記錄、月底匯出 Excel
  • 員工：用手機 LINE 每天輸入工作記錄、拍照存證
💡 使用重點
資料雲端同步後，員工手機輸入的內容會在 1~3 秒內出現在您電腦，不需要員工另外傳 JSON 備份。

3. 員工管理

員工頁籤用來建立與維護員工資料。每位員工包含員工編號、姓名、職稱、到職日、月薪、時薪、備註。

新增員工

  1. 1點「員工」頁籤
  2. 2於「新增 / 編輯員工」卡片中填寫：
    • 員工編號（可選）：公司內部編號，例如 E001
    • 姓名（必填）：員工全名
    • 職稱（可選）：如「作業員」「業務專員」
    • 到職日（可選）：用於未來年資計算
    • 月薪（底薪）（必填）：月薪新台幣
    • 時薪（可選）：留空會自動以「月薪 ÷ 30 ÷ 8」計算
    • 備註（可選）：任何補充資訊
  3. 3按綠色「儲存員工」

編輯／刪除員工

在下方員工列表找到該員工，點「編輯」載入資料修改後再按「儲存員工」覆蓋；或點「刪除」移除。

⚠ 刪除員工的影響
刪除員工只會從名單移除，不會刪除該員工已建立的每日記錄或月薪記錄。如要完全清除，需另行刪除對應記錄。

4. 產品建檔

產品建檔是論件計酬的必要前置作業。您先在這裡建立好產品名稱、預設單價和參考照片，員工輸入時只要從下拉選單選產品，單價自動帶入，減少錯誤。

開啟產品管理

在「每日」頁籤選擇「論件計酬」模式後，上方品名欄位旁邊會出現「⚙ 管理產品清單」按鈕，點它即可展開產品管理面板。

新增產品

  1. 1填寫「產品名稱」，例如 A 產品
  2. 2填寫「預設單價」，例如 50（員工選此產品時會自動帶出）
  3. 3選填「產品參考照片」：點「選擇檔案」上傳產品照片，有助於員工核對
  4. 4按「新增產品」

維護既有產品

產品清單中每一列可以：

  • 直接修改單價（輸入後自動儲存）
  • 點「📷 加照片」上傳照片，或點「✕」移除照片
  • 點「刪除」移除整個產品

  3. 從下拉選單選擇員工

  4. 選填「📷 拍照 / 上傳照片」存證

5.3 編輯／刪除每日記錄

下方「當日記錄」列表會顯示該日所有記錄，每筆右邊有兩個按鈕：

  • ✏️ 編輯 載入資料到表單，修改後按「💾 儲存修改」覆蓋
  • 刪除 移除該筆記錄

編輯時該筆記錄會顯示黃色邊框 + 「編輯中」徽章，提醒您目前正在編輯的是哪一筆。

查詢整月加總

「每日」頁籤往下滑到「整月加總」區塊，選月份和員工（留空 = 全部），會看到：

  • 每日分別的論件／計時金額
  • 每日合計
  • 月合計
  • 論件總數量、計時總時數

6. 月薪計算

月薪頁籤用來建立每月的薪資記錄，包含底薪、加班費、獎金、勞健保、扣款等項目的完整計算。適合月薪制員工或論件計酬員工的月底結算。

計算公式

  • 平日加班費 = 時薪 × 1.34 × 時數
  • 假日加班費 = 時薪 × 1.67 × 時數
  • 請假扣款 = (月薪 ÷ 30) × 請假天數
  • 應發合計 = 底薪 + 平日加班 + 假日加班 + 獎金 − 請假扣款
  • 勞保（預設）= 月薪 × 2.42%（員工負擔，可於設定頁調整費率）
  • 健保（預設）= 月薪 × 1.655%（員工負擔）
  • 扣除合計 = 勞保 + 健保 + 其他扣款
  • 實發金額 = 應發合計 − 扣除合計

操作步驟

  1. 選擇月份（如 2026-04）
  2. 選擇員工
  3. 輸入出勤天數、請假天數、平日/假日加班時數、獎金、其他扣款
  4. 勞保、健保可留空自動估算，或手動填入實際投保級距的金額
  5. 按「預覽計算」查看試算結果
  6. 確認無誤按「儲存記錄」
⚠ 勞健保建議手動填入
系統自動用月薪乘上費率估算，但實際金額應以勞保局公告的「投保薪資級距表」為準。建議查詢員工實際級距後手動輸入精確金額。

編輯／刪除月薪記錄

下方「本月薪資記錄」列表中，每筆同樣有「✏️ 編輯」和「刪除」按鈕。編輯時會自動載入所有欄位並執行預覽計算，方便調整。

7. 匯出 Excel 與照片

匯出頁籤用於產生月底報表。

匯出 Excel

  1. 點「匯出」頁籤
  2. 選擇「匯出月份」（留空 = 匯出全部資料）
  3. 按綠色「下載 Excel 檔案」按鈕

下載照片 ZIP

按「📷 下載所有照片 (ZIP)」會將當月所有員工拍攝的存證照片打包下載，檔名格式為：

檔名會與 Excel 裡「每日明細」工作表的「照片檔名」欄一致，可以直接對照核對。

💡 稽核建議
每月底同時下載 Excel 與照片 ZIP，在電腦上開啟 Excel 並解壓縮 ZIP，按「照片檔名」欄排序，可以快速對照每筆記錄與實際照片。

統計摘要

匯出頁下方有「統計摘要」表格，按月份列出人數、應發合計、實發合計，方便快速檢視。

8. 資料備份與還原

備份 JSON

匯出頁「資料管理」區塊點「備份 JSON」會下載一個包含所有員工、記錄、產品、費率設定的 JSON 檔，檔名格式 salary_backup_2026-04-20.json。

還原備份

點「還原備份」選擇先前下載的 JSON 檔，系統會讀入並覆蓋目前資料。

ℹ 雲端同步與本機備份關係
雖然資料有 Firebase 雲端同步，但建議每週仍下載一次 JSON 做雙重保險。如果雲端專案誤刪、付費方案異動，還能靠 JSON 還原。

清除全部

「清除全部」會移除本機所有資料。需要確認兩次，無法復原。

❗ 清除全部的雲端影響
清除全部只會清除本機（瀏覽器 localStorage）資料，但因為雲端訂閱仍在運作，下次重新整理頁面時雲端的資料會回來（也就是其實沒清掉）。如要真正清空，需先停用雲端同步或到 Firebase 主控台手動刪除。

9. 雲端同步

系統使用 Firebase Firestore 作為雲端資料庫，頁首下方會顯示目前同步狀態：

狀態意義
⏳ 連線中…剛啟動，正在連 Firebase
☁ 已同步 XX:XX:XX正常運作，最後同步時間
⚠ 連線失敗網路問題或設定錯誤，自動降級為只用本機

同步機制

每一次新增、編輯、刪除都會同時寫入：

  • 本機瀏覽器 localStorage（快速讀取）
  • Firebase Firestore（雲端）

其他裝置的系統會自動訂閱雲端變動，1~3 秒內同步更新畫面，不需要手動重新整理。

離線使用

沒網路時仍可正常新增記錄，資料暫存本機。恢復網路後自動補同步到雲端。

10. 系統設定

LINE LIFF 設定

設定頁籤最上方可輸入 LIFF ID，用於整合 LINE 登入。員工打開 LIFF URL 時會自動取得姓名顯示在頁首。

LIFF ID 格式：1234567890-AbCdEfGh，從 LINE Developers 主控台取得。

勞健保費率

預設值（2026 年）：

  • 勞保費率（員工負擔）：2.42%
  • 健保費率（員工負擔）：1.655%

政府調整費率時，可於此處修改並按「儲存費率」。費率會同步到雲端，所有裝置自動套用。

⚠ 費率查詢來源
請以勞動部勞工保險局、衛生福利部中央健康保險署最新公告為準。

11. 建議工作流程

初次設定（一次性）

  1. 於設定頁填入 LIFF ID 並儲存
  2. 於員工頁建立全部員工資料
  3. 於產品管理建立產品清單與照片
  4. 確認勞健保費率（視需要調整）
  5. 把 LIFF URL 分享給員工

日常運作

  • 員工每天在手機輸入工作記錄（會計不需介入）
  • 會計隨時可在電腦檢視全員資料
  • 有誤時點「編輯」修正

每週

  • 點「備份 JSON」下載備份檔，存到公司電腦指定資料夾
  • 快速瀏覽「整月加總」確認資料完整

每月月底

  1. 檢查每位員工的每日記錄是否齊全
  2. 於月薪頁建立每位員工的月薪記錄（輸入加班、獎金、扣款等）
  3. 於匯出頁：
    • 選該月份 → 下載 Excel
    • 下載照片 ZIP
    • 再下載一份 JSON 備份
  4. 用 Excel + 照片 ZIP 進行薪資發放對照與稽核

12. 常見問題

Q1. 兩台裝置資料不同步？

先檢查頁首的同步狀態，若顯示「連線失敗」可能是網路問題。重新整理網頁即可。若長時間無法連線，可能是 Firestore 規則過期（測試模式 30 天）或付費配額用完。

Q2. 員工拍照後儲存失敗？

表示本機 localStorage 快滿了（約 5-10 MB 上限）。請先下載照片 ZIP 備份後，刪除已完成月份的舊記錄以騰出空間。

Q3. 員工點 LIFF URL 顯示 400 Bad Request？

表示 Channel 狀態是「Developing」，需到 LINE Developers 切換成「Published」才能讓所有人使用。

Q4. 如何把系統搬到另一台電腦？

由於資料在雲端，新電腦只要打開同一個網址（LIFF URL 或 GitHub Pages 網址）即可自動從雲端載入資料，無需搬移。

Q5. 產品名稱打錯了怎麼辦？

進入產品管理，找到該產品點「刪除」再新增一個正確名稱的。注意已建立的記錄中的品名不會自動更新，請使用「編輯」功能個別修正。

Q6. 員工離職後怎麼處理？

建議不要直接刪除，改在員工備註欄註明「已離職 2026-05-01」。這樣歷史記錄仍可查詢，也不會影響統計報表。

Q7. Excel 裡金額是數字但排序怪怪的？

可能是 Excel 誤判為文字。開啟後選取該欄 → 右鍵「儲存格格式」→ 選「數值」即可正確排序。

Q8. 想加入新的計算欄位（例如夜班津貼）？

可以把需求告知開發端，程式可修改擴充。目前「獎金／津貼」欄位可暫代用途。

Q9. 資料被誤刪怎麼辦？

用最近的「備份 JSON」還原即可。所以強烈建議每週做一次 JSON 備份。

📞 需要協助？
使用上遇到任何問題，請聯絡系統管理員或開發端。常見調整（加欄位、改公式、換色等）都能客製修改。
